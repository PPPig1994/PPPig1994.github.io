第十章 接口、继承与多态

10.6多态

一、相关概念

  1. 向上转型

    子类型->父类型，又被称为自动类型转换

  2. 向下转型

    父类型->子类型，又被称为：强制类型转换。【需要添加类型转换符】

    无论是向上转习惯还是向下转型，两者必须是继承关系

1.Animal和Cat 之间存在继承关系，Animal是父类，cat是子类
2.Cat is a Animal【合理的】
3.new Cat()创建的对象的类型是cat，a2这个引用的数据类型是animal，可见他们进行了类型转换，子类型转换成父类型，称为向上转型或者称为自动类型转换
4.Java允许这种语法：夫类型引用指向子类型对象

二、重点内容

三、代码示例

    public class Animal {
    public void move() {
        System.out.println("动物在移动");
    }
}

public class Cat extends Animal {
    public void move() {
        System.out.println("猫走猫步!");
    }

    public void catchmouse() {
        System.out.println("猫抓老鼠");
    }
}

public class Bird extends Animal {
    public void move() {
        System.out.println("鸟在唱歌");
    }
    public void fly() {
        System.out.println("鸟在飞翔");
}
}

public class Test {

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        Animal a2 = new Cat();
        a2.move();
        // a2.catchmouse();
    }

}

四、代码分析

`Animal a2=new Cat();`

  1. Java程序永远都分为编译和运行阶段

  2. 先分析编译阶段，再分析运行阶段，编译无法通过，根本是无法运行的。

  3. 编译阶段编译器检查a2这个也能用的数据类型为Animal，由于Animal.class字节码当中有move()方法，所以编译通过了。这个国臣被我们称为静态绑定，编译器阶段绑定，只有静态绑定成功以后才有后续的运行。

  4. 在程序运行阶段，JVM堆内存当中真实创建的对象是Cat对象，那么一下程序运行阶段一定会调用Cat对象的move()方法，此时繁盛了程序的动态绑定，运行阶段绑定。

  5. 无论是cat类有没有重写move方法，运行阶段一定调用的是cat对象的move方法，因为数据真实对象是cat对象。

  6. 父类型引用指向子类型对象这种机制导致程序存在编译阶段绑定和运行阶段绑定良种不同的形态，这种机制可以成为一种多态语法机制

    a2.move();//猫在走猫步

*分析以下程序为什么不能调用？
因为编译阶段编译器检查到a2的类型是Animal类型，从Animal.class字节码文件当中无法查找到cathmouse()方法，导致静态绑定失败 *

//a2.catchmouse();

需求：
使a2对象执行cathmouse()方法
可以将a2强制类型转换为Cat类型
a2的类型是Animal（父类），转换成Cat（子类）,被称为向下转型（强制类型转换）
什么时候需要使用向下转型呢？
当调用的方法是子类型中特有的，在夫类型中不存在的，必须进行向下转型

cat c2=(cat)a2;
c2.catchmouse();//猫抓老鼠

  1. 以下的程序编译是没有问题的，因为编译器检查到的a3数据类型是Animal，与Cat直接存在继承关系，并且Animal是父类，Cat是子类，向下转型，语法合格。

  2. 程序虽让编译通过了，但是程序在运行姐u但会出现一场，因为jvm堆内存当中真实存在的兑现是Bird对象，Bird对象无法转换成cat对象，因为两种类型之间不存在继承关系，此时出现了著名的异常：

    java.lang.ClassCastException
    类型转换异常，这种异常总是在向下转型的时候会发生。

    Animal a3=new Bird();
    cat c3=(Cat)a3;

    小结：

  3. 向上转型只要通过编译，运行肯定不会出错：Animal a = new Cat()

  4. 向下转型存在隐患，编译通过但是运行可能出错：Animal a3=new Bird();cat c3=(Cat)a3;

  5. 怎样避免向下转型出现的java.lang.ClassCastException异常

  6. 使用instanceof运算符可以避免出现以上的异常

    instanceof运算符使用：

  7. 语法格式：
    （引用）instanceof （数据类型名）

  8. 执行结果类型是布尔类型，true/false

  9. 举例：假设 （a instanceof Animal）。
    true表示a这个引用指向的对象是一个Animal类型，false表示a这个引用指向的对象不是一个Animal类型

    Animal a3 = new Bird();
if (a3 instanceof Cat) {//如果a3是Cat类型
    Cat c3 = (Cat) a3;
    c3.catchmouse();
} else if (a3 instanceof Bird) {//如果a3是Bird类型
    Bird b1 = (Bird) a3;
    b1.fly();
}
输出：鸟在飞翔

五、多态的使用

多态在实际开发中的作用，以下以主任喂养宠物为实例说明多态的作用：

  1. 分析：主任喂养宠物这个场景要实现需要进行类型的抽象。

    • 主人【类】
    • 主人可以喂养宠物，所以主人有喂养这个动作
    • 宠物【类】
    • 宠物可以吃东西，所以宠物有吃东西这个动作

  2. 面向对象编程的核心：定义好类，然后将类实例化为对象，给一个环境驱使一下，让各个对象之间协作起来形成一个系统。

  3. 多态的作用是什么？
    降低程序的耦合度，提高程序的扩展力，因此尽可能使用多态，父类型引用指向子类型对象

        /**
     * Master类 
     * @author luffy Master 和Cat、Dog这两个类型的关联程度很强，耦合度高，扩展能力差
     */
    // 降低城的的耦合度【解耦合】，提高程序的扩展了【软件开发的一个很重要的目标】
    public class Master {
        // 不使用多态语法机制
    public void feed(Cat cat) {
        cat.eat();
    }

    public void feed(Dog dog) {
        dog.eat();
    }
    // 使用多态语法机制

    // Master主人面向的是一个抽象的Pet，不再是具体的宠物
    // 提倡：面向对象编程，不要面向具体编程
    // 面向抽象编程的好处是：耦合度低，扩展力强
    public void feed(Pet pet) {
        pet.eat();
    }
}
public class Pet {
    public void eat() {
        System.out.println("动物在吃饭");
    }
}
public class Cat extends Pet {
    public void eat() {
        System.out.println("猫在吃");
    }
}
public class Dog extends Pet {
    public void eat() {
        System.out.println("狗在吃");
    }
}
public class Test {

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        Master master = new Master();
        // 不使用多态语法机制
        Cat cat = new Cat();
        cat.eat();
        Dog dog = new Dog();
        dog.eat();
        // 使用多态语法机制
        master.feed(new Cat());
        master.feed(new Dog());
    }

}
输出：猫在吃
     狗在吃
     猫在吃
     狗在吃
